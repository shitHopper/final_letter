# 绝笔信 — 技术选型对比

## 一、前端框架（手机端）

| 维度 | React Native | Flutter | UniApp | 原生开发 (Kotlin/Swift) |
|------|-------------|---------|--------|------------------------|
| **开发语言** | JavaScript/TypeScript | Dart | Vue/JavaScript | Kotlin + Swift |
| **跨平台** | iOS + Android | iOS + Android | iOS + Android + 小程序 + H5 | 仅单平台，需写两套 |
| **开发效率** | 高 | 高 | 最高 | 低 |
| **性能** | 中高 | 高（接近原生） | 中 | 最高 |
| **UI一致性** | 中 | 高 | 中 | 各平台原生风格 |
| **社区生态** | 非常成熟 | 成熟且快速增长 | 国内生态好 | 各自成熟 |
| **适合场景** | 通用App | 对UI和动画要求高 | 需要小程序+App同时覆盖 | 对性能极致要求 |
| **学习成本** | 低（JS生态） | 中（需学Dart） | 低（Vue生态） | 高（两套语言） |
| **国内推广** | 一般 | 好 | **最好**（可同时出微信小程序） | 一般 |
| **推荐指数** | ★★★★ | ★★★★★ | ★★★★ | ★★★ |

**推荐：Flutter** — 性能好、UI一致性强、一套代码两端运行、社区活跃

## 五、整体方案推荐

### 方案A：快速启动型（推荐）

```
前端：Flutter
后端：NestJS + PostgreSQL
邮件：Resend / 阿里云邮件推送
短信：阿里云短信
实体邮信：二期实现，先预留接口
部署：Docker + 云服务器（阿里云/腾讯云）
```

- 优点：开发快、成本低、团队小即可运转
- 缺点：Go级别的极致性能需后续优化

### 方案B：全栈国内云

```
前端：UniApp（同时出App+小程序）
后端：NestJS + MySQL
邮件：阿里云邮件推送
短信：阿里云短信
实体邮信：二期
部署：阿里云全家桶
```

- 优点：国内生态完整、小程序利于推广
- 缺点：UniApp性能不如Flutter

### 方案C：企业级重型

```
前端：原生开发（Kotlin + Swift）
后端：Spring Boot + PostgreSQL
邮件：SendGrid
短信：Twilio
实体邮信：PostGrid
部署：K8s + 多云
```

- 优点：性能最强、扩展性最好
- 缺点：开发成本高、周期长、需多人团队

---

## 六、推荐结论

**建议采用方案A（Flutter + NestJS + PostgreSQL）**，理由：

1. **Flutter** 性能接近原生，UI一致性好，一套代码双端运行
2. **NestJS** 前后端统一用 TypeScript，降低沟通成本，开发效率高
3. **PostgreSQL** 功能全面，社区内容搜索和用户数据关系都能覆盖
4. 推送服务先用阿里云体系，国内送达率有保障
5. 实体邮信业务复杂，建议MVP阶段预留接口，二期再对接
